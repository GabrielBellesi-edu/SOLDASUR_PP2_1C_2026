# Cambios en Versión v5.0.2

Esta versión introduce mejoras arquitectónicas clave para la unificación del diseño de constantes de sistemas expertos, una convención de nombres limpia y la automatización del proceso de escalamiento de nuevas marcas con calculadoras y logos.

## Resumen de Modificaciones

### 1. Unificación de Constantes y Nombres de Sistemas Expertos
Para lograr una arquitectura simétrica y ordenada, se implementó una convención de nombres unificada para todos los archivos de motores y bases de conocimiento de sistemas expertos:
*   **Archivos de Motores:** `expert_engine_{marca}.py`
    *   `expert_engine.py` $\rightarrow$ `expert_engine_peisa.py` (PEISA)
    *   `weber_expert_engine.py` $\rightarrow$ `expert_engine_weber.py` (Weber)
*   **Bases de Conocimiento (KB):** `advisor_knowledge_base_{marca}.json`
    *   `peisa_advisor_knowledge_base.json` $\rightarrow$ `advisor_knowledge_base_peisa.json` (PEISA)
    *   `weber_advisor_knowledge_base.json` $\rightarrow$ `advisor_knowledge_base_weber.json` (Weber)
*   **Embebido de Constantes (PEISA):** Se reubicó el diccionario estático `RADIATOR_MODELS` directamente dentro de `app/modules/expertSystem/expert_engine_peisa.py`, eliminando la dependencia de carga externa a través de `models.py`.
*   **Eliminación de Acoplamientos:** Se actualizaron las importaciones de `RADIATOR_MODELS` en todos los archivos relacionados (`app/app.py`, `app/main.py` y `app/test_import.py`) para referenciar la nueva ubicación unificada.
*   **Depreciación de `app/models.py`:** Se vació y deprecó el archivo `app/models.py`, dejando únicamente comentarios de redirección.

### 2. Automatización del Instalador de Marcas (`install_brand.py`)
*   **Soporte para Logotipos:** El instalador busca automáticamente archivos que contengan la palabra `logo` (ej: `marca-logo.png`, `logo.png`) en el directorio de origen y los copia a `web_app/img/{brand_lower}-logo.png`. De esta manera, el frontend puede mostrar el logo en los botones de búsqueda y en el catálogo sin cambios manuales.
*   **Copia de Sistemas Expertos Unificada:** Si el archivo `config.json` de la marca origen contiene `"has_calculator": true`, el instalador verifica y copia automáticamente:
    *   `calculator_kb.json` $\rightarrow$ `app/advisor_knowledge_base_{brand_lower}.json`
    *   `calculator_engine.py` $\rightarrow$ `app/modules/expertSystem/expert_engine_{brand_lower}.py`
*   **Registro Dinámico de Calculadoras:** Se actualizó el generador del archivo `configs/brands_registry.json` para registrar dinámicamente `"has_calculator"` y `"calculator_display"` a partir de los datos configurados en el `config.json` de la marca origen.

### 3. Rutas Dinámicas para Sistemas Expertos en `app/main.py`
*   Se refactorizaron los endpoints `/api/expert/{brand}/start` y `/api/expert/{brand}/reply`.
*   Se implementó un cargador dinámico de motores expertos (`get_expert_engine`) que:
    1.  Carga el motor de cálculo personalizado (`expert_engine_{brand_lower}.py`) utilizando `importlib` si existe en el proyecto.
    2.  Si no hay motor personalizado, utiliza de forma automática una instancia genérica de `ExpertEngine` alimentada con la base de conocimiento JSON unificada (`advisor_knowledge_base_{brand_lower}.json`).
    3.  Mantiene Weber como flujo legado administrado localmente para preservar su lógica estructurada específica.

### 4. Unificación de Nombres para RAG y Bases de Datos Vectoriales
Para mantener una simetría total con el esquema de nombres de los sistemas expertos, se unificaron los nombres de los archivos del buscador semántico (RAG) y sus bases de datos vectoriales utilizando la convención de prefijo primero:
*   **Weber RAG y BD:**
    *   `weber_rag_query.py` $\rightarrow$ `rag_query_weber.py`
    *   `weber_rag_llm.py` $\rightarrow$ `rag_llm_weber.py`
    *   `weber_products.faiss` $\rightarrow$ `products_weber.faiss`
    *   `weber_metadata.json` $\rightarrow$ `metadata_weber.json`
*   **PEISA RAG y BD:**
    *   `peisa_rag_query.py` $\rightarrow$ `rag_query_peisa.py`
    *   `peisa_rag_llm.py` $\rightarrow$ `rag_llm_peisa.py`
    *   `peisa_products.faiss` $\rightarrow$ `products_peisa.faiss`
    *   `peisa_products.db` $\rightarrow$ `products_peisa.db`
*   **Actualización de Importaciones y Rutas:**
    *   Se actualizaron las importaciones de RAG en `app/main.py`, `app/orchestrator.py` y `configs/brands_registry.json`.
    *   Se modificó el script de inicialización de marcas `install_brand.py` para generar los nuevos nombres de archivos (`products_{marca}.faiss`, `metadata_{marca}.json`, `rag_query_{marca}.py`, `rag_llm_{marca}.py`) y registrar el módulo RAG correspondiente con el nuevo formato en el archivo de configuración global.
