# Documentación de Cambios: Versión 3 (v3) → Versión 4 (v4)

Este documento detalla la evolución del sistema **SOLDASUR** desde la versión v3 hacia la versión v4. El foco principal de esta actualización ha sido la **modularización** del código JavaScript y la **migración a frontend** (cliente) de la calculadora del sistema experto de Weber, eliminando su dependencia del backend en Python para esta tarea.

---

## 1. Objetivos del Paso a la v4

1. **Calculadora Weber Offline (Frontend-Only):**
   * En la v3, la calculadora Weber requería que el servidor de FastAPI (Uvicorn) estuviera corriendo para procesar el árbol de decisión (JSON) y calcular los consumos en Python (`weber_expert_engine.py`).
   * En la v4, toda la lógica de Weber (preguntas, flujo de navegación y cálculos matemáticos de materiales) se ejecuta **100% localmente en el navegador**, imitando el comportamiento offline e independiente que ya tenía la calculadora de PEISA.
2. **Modularización y Limpieza de Código:**
   * En la v3, el código de interacción con el chat, renderizado, catálogos y sistemas expertos estaba acoplado y duplicado en archivos monolíticos.
   * En la v4, el código se divide en submódulos ordenados dentro de `app/modules/`, permitiendo que cada archivo tenga una única responsabilidad.
3. **Orquestación Limpia:**
   * `soldasur.js` se simplifica para actuar únicamente como el inicializador del menú y orquestador del flujo principal.

---

## 2. Nueva Estructura Modular de JavaScript

Se ha reestructurado la distribución de responsabilidades en la carpeta del frontend, quedando de la siguiente forma:

```text
SOLDASUR_PP2_1C_2026/
│
├── app/
│   └── modules/
│       ├── core/
│       │   └── core.js            <-- [NUEVO] Estado común, UI helpers y catálogo
│       ├── peisa/
│       │   └── peisa_expert.js    <-- [NUEVO] Sistema experto y calculadora PEISA
│       ├── weber/
│       │   └── weber_expert.js    <-- [NUEVO] Sistema experto y calculadora Weber (Frontend)
│       └── chatbot/
│           └── chatbot.js         <-- Chat interactivo IA (requiere Ollama/Backend)
│
├── soldasur.js                     <-- Orquestador principal (menú y ruteo inicial)
└── index.html                      <-- Punto de entrada (actualizado orden de carga de scripts)
```

## 3. Detalle de Cambios por Archivo

A continuación se describen los archivos nuevos y modificados de la v4 en relación con la v3, utilizando enlaces relativos para facilitar la navegación en el repositorio:

### 3.1. Archivos Nuevos

#### 📂 [core.js](../app/modules/core/core.js)
Este archivo centraliza el núcleo de la aplicación, liberando a los demás scripts de declarar variables globales duplicadas y funciones utilitarias repetitivas.
* **Estado Global:** Agrupa variables del chat como `conversationId`, `chatIsOpen`, `isMaximized`, `currentMode`, entre otras.
* **Helpers de Interfaz (UI):** Funciones de manipulación del DOM como `appendMessage()`, `renderOptions()`, `scrollToBottom()`, y los inputs de usuario de tipo numérico (`createNumberInput()`) y de dimensiones (`createDimensionsInput()`).
* **Catálogo y Sucursales:** Gestión de consulta de sucursales (`consultSucursal()`) y visualización de productos en el chat (`loadProductCatalog()`, `showCategoryMenu()`, `showProductsByCategory()`, `showAllProducts()`).

#### 📂 [peisa_expert.js](../app/modules/peisa/peisa_expert.js)
Contiene la lógica matemática y de recomendación del sistema experto de calefacción **PEISA**.
* **Cuestionario Guiado:** Manejo de preguntas sobre ambientes, medidas y tipo de calefacción.
* **Lógica de Cálculo:** Cálculos térmicos locales en JavaScript (`calculateHeatingLoad()`, `calculateRadiatorLoad()`, `calculateBoilerLoad()`).
* **Visualización de Recomendados:** Muestra de forma interactiva en tarjetas de producto los radiadores, calderas o toalleros recomendados para la obra.

#### 📂 [weber_expert.js](../app/modules/weber/weber_expert.js)
Este archivo representa el cambio clave de la v4: el motor experto de **Weber** portado por completo a JavaScript.
* **Árbol de Decisión Embebido:** Se tradujo el archivo JSON de preguntas (`weber_advisor_knowledge_base.json`) a una constante estructurada de JavaScript (`WEBER_FLOW`), permitiendo la navegación interactiva inmediata sin llamadas de red (`fetch`).
* **Conversión de Cálculos:** La función `calcularMaterialesWeber(soporte, m2)` replica con exactitud los cálculos que antes hacía el script Python `resolver_calculo()`, incluyendo rendimientos por tipo de soporte y un 10% adicional de desperdicio.
* **Renderizado Local:** Funciones `iniciarExpertoWeber()`, `renderWeberNode()` y `enviarRespuestaWeber()` que administran la UI del chat de manera reactiva e independiente de cualquier backend.

---

### 3.2. Archivos Modificados

#### 📂 [soldasur.js](../soldasur.js)
* **Antes (v3):** Actuaba como un contenedor monolítico que mezclaba lógica de inicialización, control de vistas, catálogo de productos, lógica de sucursales y manejo de PEISA.
* **Ahora (v4):** Reducido a unas ~60 líneas. Únicamente expone la función de inicialización `startConversation()` y el enrutador de opciones principales `handleOptionClick()`, delegando el resto del procesamiento a `core.js`, `peisa_expert.js` y `weber_expert.js`.

#### 📂 [index.html](../index.html)
* **Antes (v3):** Cargaba scripts de manera directa, resultando en dependencias implícitas no controladas.
* **Ahora (v4):** Se actualizó el orden en el que se importan los scripts JavaScript al final del `<body>` para asegurar que las clases, funciones globales y de UI estén disponibles cuando los scripts específicos de PEISA, Weber, el Chatbot e inicialización lo requieran:
  ```html
  <!-- Scripts Modulares -->
  <script src="app/modules/core/core.js"></script>
  <script src="app/modules/peisa/peisa_expert.js"></script>
  <script src="app/modules/weber/weber_expert.js"></script>
  <script src="app/modules/chatbot/chatbot.js"></script>
  <script src="soldasur.js"></script>
  ```

#### 📂 [chatbot.js](../app/modules/chatbot/chatbot.js)
* **Modificación:** Se eliminaron las declaraciones de variables globales como `peisaProductsFromJSON` y funciones como `loadProductCatalog()`, las cuales pasaron a estar unificadas bajo el archivo común `core.js`. Esto soluciona conflictos de redeclaraciones en el scope global del navegador al ejecutarse en conjunto.

---

## 4. Archivos que NO sufrieron cambios

Es importante notar que para mantener compatibilidad y la posibilidad de futuras expansiones:
1. **Backend Python (`app/`):** Todos los archivos Python (`main.py`, `weber_expert_engine.py`, etc.) siguen presentes en la estructura del proyecto v4 sin modificaciones. El chatbot de lenguaje natural de `chatbot.js` sigue utilizando el servidor Uvicorn de backend si está disponible.
2. **Estilos (`soldasur.css`):** La hoja de estilos generales y de interfaz de chat permanece idéntica a la v3.

---

## 5. Beneficios Técnicos y Operativos de la v4

* **Resiliencia Operativa:** Las calculadoras (PEISA y Weber) funcionan de forma offline/local en su totalidad. El usuario no recibirá el molesto error de *"Verificá que el backend Uvicorn esté corriendo"* al intentar calcular materiales de construcción.
* **Mantenibilidad:** El código está desacoplado. Agregar un nuevo tipo de producto o un tercer sistema experto (ej: una calculadora de pintura) consiste simplemente en crear un módulo nuevo en `app/modules/` e importarlo en `index.html`, sin riesgo de alterar el funcionamiento del núcleo existente.
* **Rendimiento:** Al no requerirse llamadas HTTP de red local para cada paso del cuestionario Weber ni para el procesamiento matemático, el flujo del usuario es instantáneo, brindando una experiencia de navegación mucho más fluida.
