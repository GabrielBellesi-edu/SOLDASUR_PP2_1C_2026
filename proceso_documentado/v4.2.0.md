# Bitácora de Cambios v4.2.0 — Reestructuración Modular, Simetría RAG y Unificación de Catálogos

Esta bitácora documenta las mejoras y cambios realizados en la versión **v4.2.0** de **SOLDASUR** para lograr modularidad, simetría total en la arquitectura RAG y limpieza del repositorio.

---

## 1. Cambios Clave Implementados

### A. Estructura de Carpetas Modularizada
* **Frontend Limpio (`web_app/`):** Todo el código del frontend que corre en el navegador fue reubicado en la carpeta `web_app/`. Los módulos JavaScript se encuentran en `web_app/js_modules/`, las imágenes en `web_app/img/`, y los catálogos en `web_app/data/`.
* **Motor RAG Consolidado (`RAG_engine/`):** Se centralizó la lógica RAG del proyecto. Las bases de datos e índices vectoriales (SQLite/FAISS) residen en `RAG_engine/database/`. Las consultas e inferencia LLM en `RAG_engine/query/`, y los scripts de ingesta en `RAG_engine/scripts/`.
* **Separación de Scraping (`scraping/`):** Se aislaron los scripts de obtención de catálogos comerciales (`peisa_product_scraper.py` y `weber_product_scraper.py`) en una sección dedicada para simplificar su mantenimiento.

### B. Simetría RAG en Weber y PEISA
* **División Weber:** Al igual que PEISA, la lógica de Weber fue dividida en dos componentes especializados:
  * **Retriever (`weber_rag_query.py`):** Encargado de cargar el índice FAISS y recuperar los productos semánticamente similares a la consulta del usuario.
  * **Generator (`weber_rag_llm.py`):** Llama a Ollama y genera la respuesta contextual del chatbot "Soldy".
* **Orquestador Central de Ingesta:** Se implementó `RAG_engine/scripts/ingest.py` como único punto de entrada para recrear las bases vectoriales mediante flags:
  * `python RAG_engine/scripts/ingest.py --peisa` (ejecuta `peisa_ingest.py` leyendo `peisa_catalog.json`)
  * `python RAG_engine/scripts/ingest.py --weber` (ejecuta `weber_build_catalog.py` y `weber_build_embeddings.py`)
  * `python RAG_engine/scripts/ingest.py --all` (ejecuta la ingesta completa)

### C. Unificación de Catálogos y Rutas
* **Catálogos Coherentes:** Se renombró el catálogo comercial de PEISA de `products_catalog.json` a `peisa_catalog.json`, logrando consistencia con `weber_catalog.json`. Ambos se alojan bajo `web_app/data/`.
* **Actualización del Servidor FastAPI:** Se configuró `app/main.py` para montar la raíz estática en `web_app/` y enrutar las peticiones utilizando las nuevas ubicaciones del RAG y catálogos.
* **Actualización del Frontend JS:** Se modificó `web_app/js_modules/core.js` para realizar el fetch a `peisa_catalog.json` y mapear las imágenes de los logos a la nueva carpeta `web_app/img/`.

```text
├── docs/                      # Manuales de arquitectura y RAG

├── scraping/                  # Scripts de scrapers (peisa_product_scraper.py, weber_product_scraper.py)
├── RAG_engine/                # Motor RAG y almacenamiento vectorial
│   ├── database/              # SQLite e índices FAISS (peisa_products.db, peisa_products.faiss, etc.)
│   ├── scripts/               # Ingestor central y scripts de lógica por marca
│   └── query/                 # Módulos de búsqueda e inferencia LLM por marca
└── Legacy/                    # Archivos históricos inactivos
```
