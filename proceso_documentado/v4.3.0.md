# Bitácora de Extracción de Metadatos y Taxonomías Estructuradas — SOLDASUR v4.3.0

Esta bitácora documenta las especificaciones técnicas, el diseño y las modificaciones realizadas para la versión **v4.3.0** del Asistente Técnico Inteligente de SOLDASUR.

## 1. Motivación y Diagnóstico
En versiones anteriores de scraping y RAG para Weber:
- La información de filtros del buscador oficial de Weber (Área, Actividades, Colores, Ancho de junta, Terminación, etc.) se perdía porque el scraper solo guardaba un bloque de texto plano e inestructurado.
- El catálogo unificado de productos dependía puramente de heurísticas de expresiones regulares (regex) inestables sobre el texto plano para inferir el "Soporte" o la "Presentación", lo cual generaba inconsistencias y pérdida de información.

**Solución:** Se analizó estructuralmente el DOM de las páginas guardadas del buscador y de productos individuales. Se identificó que cada página de producto individual contiene los metadatos y etiquetas directamente embebidos de manera parseable:
- **Taxonomías/Filtros:** Enlaces a rutas `/search-content/content_type/product/...` organizados en la columna derecha.
- **Especificaciones técnicas:** Encabezados `<h2>` con el nombre del atributo, seguidos por un párrafo `<p>` con el valor del atributo (ej. `<h2>Soporte</h2><p>Piso</p>`).

## 2. Archivos Modificados e Incorporados

* **[MODIFY] [weber_product_scraper.py](../scraping/weber_product_scraper.py):**
  - Implementada limpieza del DOM para aislar el contenedor principal `<main>` del producto.
  - Diseñado extractor de enlaces de taxonomía mapeando categorías como `product_benefits` -> `beneficios`, `activities` -> `actividades`, `area` -> `area`, etc.
  - Implementado extractor genérico de atributos técnicos en formato `h2` + `p`.
  - Guardado de campos `"filtros"` y `"atributos_tecnicos"` en el JSON individual del producto.
  - Configurada reconfiguración automática de `stdout` a UTF-8 en Windows para evitar fallos de codificación con emojis.

* **[MODIFY] [weber_build_catalog.py](../RAG_engine/scripts/weber_build_catalog.py):**
  - Actualizado `build_product_entry` para priorizar los datos estructurados `"filtros"` y `"atributos_tecnicos"` sobre el análisis heurístico por regex del texto plano.
  - Incorporados nuevos campos al catálogo consolidado `weber_catalog.json` (como `colores`, `filtros` y `atributos_tecnicos`).

* **[MODIFY] [weber_build_embeddings.py](../RAG_engine/scripts/weber_build_embeddings.py):**
  - Actualizada la función `build_chunk_text` para que incluya de forma explícita los colores en el texto que se le suministra a SentenceTransformers.
  - Incluidos los campos `colores`, `filtros` y `atributos_tecnicos` en los metadatos finales del índice RAG (`weber_metadata.json`).

## 3. Verificación de Funcionamiento

1. **Test del Scraper Local (Playwright):**
   Se verificó la extracción de los 100 productos del buscador y el procesamiento detallado de 2 productos con éxito:
   ```powershell
   python scraping/weber_product_scraper.py --max-productos 2 --sin-pdfs
   ```
   *Resultado:* Los JSONs individuales (`weber_ras.json`, `weber_top.json`) se crearon con la estructura correcta de filtros y atributos técnicos (ej. `Soporte: Piso`).

2. **Compilación del Catálogo:**
   Se compiló el catálogo consolidado:
   ```powershell
   python RAG_engine/scripts/weber_build_catalog.py
   ```
   *Resultado:* Generado `web_app/data/weber_catalog.json` con 119 productos normalizados e integrados.

3. **Generación de Embeddings RAG:**
   Se generó el índice vectorial FAISS:
   ```powershell
   venv\Scripts\python RAG_engine/scripts/weber_build_embeddings.py
   ```
   *Resultado:* Actualizados con éxito `weber_products.faiss` y `weber_metadata.json` con 119 vectores indexados que contienen los nuevos campos para enriquecer las respuestas de Soldy.
