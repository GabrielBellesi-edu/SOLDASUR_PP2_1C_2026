# Documentación de la Versión v5.0.1: Galería Dinámica, Menú Dinámico y Sistema Experto Genérico

Esta versión introduce una arquitectura desacoplada para el chat inteligente de **SOLDASUR**. Todas las marcas, categorías de productos y sistemas expertos (calculadoras) se cargan y ejecutan de manera 100% dinámica a través del servidor FastAPI en el backend.

---

## 1. Menú Principal y Saludo de Soldy

El saludo inicial de Soldy ha sido actualizado para reflejar claramente las capacidades del asistente técnico:
> *"¡Hola! Soy Soldy, tu asistente inteligente de SOLDASUR. Puedo ayudarte a buscar productos por marca, a hacer algún cálculo o a responder a alguna pregunta. ¿Qué necesitás?"*

El menú principal ahora presenta exactamente dos opciones estructuradas:
*   **`Calculadoras`**: Despliega un submenú con todas las calculadoras activas de las marcas configuradas (PEISA, Weber, u otras futuras marcas que tengan sistema experto).
*   **`Buscar productos`**: Accede a la navegación por catálogo, mostrando las marcas registradas y sus respectivas categorías de manera dinámica.

---

## 2. Arquitectura de Sistema Experto Genérico (Calculadoras)

En lugar de implementar lógica de cuestionarios y cálculos matemáticos directamente en JavaScript en el frontend, v5.0.1 centraliza la toma de decisiones y cálculos en el backend:

### Flujo de Comunicación Cliente-Servidor:
1.  **Inicio:** Al seleccionar una calculadora, el frontend instancia la clase `GenericExpertClient(brandKey)` y hace un `POST /api/expert/{brand}/start`.
2.  **Cuestionario Dinámico:** El backend responde con el primer nodo de la base de conocimiento JSON, que especifica:
    *   La pregunta a realizar.
    *   El tipo de control a renderizar (opciones fijas, campo numérico simple o campos numéricos múltiples).
3.  **Respuestas:** El usuario interactúa y el cliente envía la respuesta mediante `POST /api/expert/{brand}/reply` con el índice seleccionado o los valores ingresados.
4.  **Cálculo:** El backend realiza las operaciones matemáticas usando la lógica de negocio en Python (`resolver_calculo()`) y devuelve el nodo final con el resumen y los productos recomendados.

---

## 3. Catálogo y Galería de Productos 100% Dinámica

*   **API `/api/brands`:** Retorna el registro central de marcas registradas.
*   **API `/api/brands/{brand_key}/catalog`:** Carga el catálogo JSON correspondiente a la marca de forma dinámica.
*   **Navegación:** `core.js` agrupa y muestra las categorías basándose en los datos recuperados en tiempo real. Soporta la renderización de imágenes locales (`imagen_local`) y remotas (`imagen_url`) con fallback automático a logos genéricos.

---

## 4. Guía para Agregar Nuevas Marcas (Escalamiento)

Para agregar una nueva marca al sistema sin modificar el código del frontend:

### Paso 1: Crear Catálogo y Configuración
Crea una carpeta para tu marca dentro de `nueva_marca/` (por ejemplo, `nueva_marca/Tigre/`). Debe contener:
*   `config.json`: Metadatos, palabras clave y prompts.
*   `catalog.json` (o `marca_catalog.json`): Listado de productos con sus campos (`model`, `category`, `description`, `imagen_url`, `url`).

### Paso 2: Ejecutar el Instalador
Ejecuta `instalar_marca.bat` y escribe el nombre de la carpeta (ej: `Tigre`). El script automatiza:
*   La compilación del índice vectorial FAISS para el RAG.
*   El registro de la marca en `configs/brands_registry.json` con sus banderas:
    ```json
    "has_calculator": false,
    "calculator_display": "Calculadora Tigre"
    ```

### Paso 3: (Opcional) Agregar una Calculadora
Si la nueva marca requiere una calculadora:
1.  Establece `"has_calculator": true` en su registro en `brands_registry.json`.
2.  Crea la base de conocimientos del cuestionario en `app/{brand_lower}_advisor_knowledge_base.json`.
3.  Implementa la lógica del motor experto en `app/modules/expertSystem/{brand_lower}_expert_engine.py`.
4.  Vincúla la marca en `app/main.py` dentro de los endpoints `/api/expert/{brand}/start` y `/api/expert/{brand}/reply`.
